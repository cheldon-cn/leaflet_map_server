# ArcGIS Pro 工程迁移系统详细设计文档

## 1. 系统概述

### 1.1 设计目标
开发一套稳健的 ArcGIS Pro 工程迁移系统，支持 2.x 和 3.x 版本 `.aprx` 文件的一键迁移，完整同步数据、地图和符号系统到 QGIS 3.34 LTS 环境。

### 1.2 核心要求
- **效率要求**：单个工程迁移时间 ≤ 30 秒
- **批量处理**：支持 200+ 工程批量处理
- **架构解耦**：数据复制与工程转换解耦
- **环境分离**：提供中间 XML 格式用于无 Pro 环境重建
- **正确性保证**：通过自动化验证确保迁移质量

### 1.3 技术栈
- **开发语言**：Python
- **核心框架**：ArcPy（读取阶段）、PyQGIS（重建阶段）
- **并行处理**：QgsTask 框架
- **数据库工具**：PostgreSQL 逻辑复制、pg_dump/pg_restore
- **中间格式**：基于 XSD 的 XML 定义

## 2. 工程解析模块

### 2.1 功能描述
- 基于 ArcPy 解析 `.aprx` 工程文件
- 提取地图、图层、布局、符号等核心元素
- 生成符合 XML Schema 的中间文档
- 实现数据采样机制（前 N 条记录）

### 2.2 输入输出定义
| 项目 | 描述 |
|------|------|
| **输入** | `.aprx` 工程文件、配置参数（采样数量 N、输出路径等） |
| **输出** | 中间 XML 文件、日志文件、错误报告 |
| **输出格式** | 符合 XSD 定义的 XML，包含完整工程元数据、数据采样记录 |

### 2.3 关键算法/流程
1. **工程初始化**
   - 使用 `arcpy.mp.ArcGISProject()` 加载 `.aprx` 文件
   - 验证工程版本（2.x/3.x 兼容性）

2. **元数据提取**
   - 提取工程属性：名称、版本、保存时间、主文件夹
   - 遍历所有地图、布局、图层、表格

3. **图层信息收集**
   - 对每个图层：
     a. 提取数据源信息（类型、路径、连接字符串）
     b. 获取 CRS 并转换为完整 WKT
     c. 提取字段定义（名称、类型、长度、精度）
     d. 获取符号系统（CIM JSON）
     e. 提取标注设置、定义查询、透明度等属性

4. **数据采样**
   - 对每个图层执行查询获取前 N 条记录（N 默认=10，可配置）
   - 序列化属性值（排除 BLOB 字段）
   - 将几何对象转换为 WKT/WKB 格式
   - 记录要素总数

5. **XML 序列化**
   - 按照 XSD 结构组织数据
   - 生成包含工程元数据、地图列表、图层定义、布局定义、样式库、脚本清单的 XML

### 2.4 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 单个工程解析时间 | ≤ 10 秒 | 从加载 `.aprx` 到生成 XML 的时间 |
| 内存占用 | ≤ 500 MB | 监控进程内存使用峰值 |
| 采样效率 | 1000 要素/秒 | 采样 N 条记录的时间 |
| 错误率 | < 1% | 解析失败的工程比例 |

### 2.5 异常处理方案
1. **工程加载失败**
   - 检查文件路径有效性
   - 验证 ArcGIS Pro 版本兼容性
   - 记录详细错误信息并跳过

2. **数据源不可访问**
   - 网络/数据库连接失败时重试机制（最多3次）
   - 标记为"数据源不可用"状态
   - 继续处理其他可访问图层

3. **采样过程异常**
   - 处理大表时设置查询超时（默认30秒）
   - 采样失败时记录原因并继续
   - 提供部分采样数据或跳过采样

4. **XML 生成错误**
   - 验证数据完整性
   - 使用 XML Schema 验证
   - 生成错误报告并回滚

### 2.6 测试用例
| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 正常工程解析 | 成功生成完整 XML | 验证 XML 符合 XSD，包含所有地图图层 |
| 大工程文件（>100 图层） | 在 10 秒内完成解析 | 计时测试，内存监控 |
| 损坏的 .aprx 文件 | 优雅失败，生成错误报告 | 检查错误日志和状态码 |
| 无权限数据源 | 标记不可访问状态，继续处理 | 验证 XML 中数据源状态字段 |
| 采样数量配置 | 按配置 N 值采样记录 | 验证 XML 中采样记录数量 |

## 3. 数据迁移模块

### 3.1 功能描述
- 处理本地数据（shp 等）到 GeoPackage 的转换
- 实现 PostgreSQL 数据两种复制策略
- 开发表路由系统基于外部映射表
- 处理空间函数/视图的迁移方案

### 3.2 输入输出定义
| 项目 | 描述 |
|------|------|
| **输入** | 中间 XML 文件、数据库连接配置、路由映射表 |
| **输出** | 迁移后的数据文件/数据库、状态报告、错误日志 |
| **本地输出** | GeoPackage 文件、复制的 Shapefile |
| **数据库输出** | 目标 PostgreSQL 数据库中的表结构+数据 |

### 3.3 关键算法/流程
1. **数据源分类**
   - 解析 XML 中的图层数据源信息
   - 分类为：本地文件、PostgreSQL 表、其他数据库

2. **本地数据处理**
   - Shapefile 到 GeoPackage 转换：
     a. 使用 OGR/GDAL 进行格式转换
     b. 保留字段类型、空间参考
     c. 处理中文字符编码
   - 文件夹结构重建：
     a. 根据原始工程结构创建目录
     b. 复制或转换文件到对应位置

3. **PostgreSQL 数据迁移**
   - **策略选择**：
     a. 小表（<100GB）：使用 pg_dump/pg_restore 或导出为 GeoPackage
     b. 大表（≥100GB）：标记为逻辑复制
   - **连接字符串路由**：
     a. 查询外部映射表获取目标连接信息
     b. 支持基于 schema.table 的复杂路由规则
     c. 生成最终连接字符串写入 XML

4. **逻辑复制配置**
   - 为超大表生成逻辑复制配置：
     a. 创建发布（PUBLICATION）
     b. 配置复制槽
     c. 设置过滤条件
     d. 在 XML 中嵌入配置参数

5. **数据库对象处理**
   - **空间函数/视图**：
     a. 提取函数定义
     b. 分析依赖的 PostGIS 函数
     c. 生成迁移报告或自动转换
   - **触发器**：
     a. 提取触发器定义
     b. 验证目标环境兼容性
     c. 按策略处理（报告/自动）

### 3.4 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 本地文件转换速度 | ≥ 10 MB/秒 | 监控文件转换吞吐量 |
| PostgreSQL 小表迁移 | ≤ 5 秒/GB | 从开始导出到导入完成的时间 |
| 逻辑复制初始同步 | 最大化网络带宽利用率 | 监控同步进度和速度 |
| 并发处理能力 | 支持 10 个并行迁移任务 | 测试同时处理多个表 |

### 3.5 异常处理方案
1. **文件转换失败**
   - 检查源文件完整性
   - 验证目标路径可写
   - 尝试修复或跳过损坏文件

2. **数据库连接异常**
   - 实现连接池和重试机制
   - 验证连接参数
   - 分级错误处理：网络问题 vs 权限问题

3. **pg_dump/pg_restore 失败**
   - 监控进程状态和输出
   - 解析错误信息
   - 自动重试（可配置次数）
   - 回滚部分完成的操作

4. **逻辑复制配置错误**
   - 验证 PostgreSQL 版本支持
   - 检查 wal_level 设置
   - 测试发布/订阅配置
   - 提供详细配置指南

5. **空间函数不兼容**
   - 记录不兼容的函数列表
   - 提供替代方案建议
   - 生成 DBA 审核报告

### 3.6 测试用例
| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| Shapefile 转 GeoPackage | 数据完整转换，属性无损 | 对比源和目标数据记录 |
| 小表 pg_dump/restore | 表结构和数据完全复制 | 行数对比、字段类型验证 |
| 连接字符串路由 | 正确解析目标数据库 | 验证最终连接字符串 |
| 超大表逻辑复制标记 | XML 中标记"建议逻辑复制" | 检查 XML 状态字段 |
| 并发数据迁移 | 多个表同时迁移，无冲突 | 监控进程状态和数据一致性 |

## 4. 样式转换模块

### 4.1 功能描述
- 解析 `.stylx` 文件，提取符号定义
- 开发转换器将 ArcGIS 符号转为 QGIS 格式
- 集成开源样式转换工具（如 SLYR 社区版）
- 生成 QGIS 样式库文件（.xml）

### 4.2 输入输出定义
| 项目 | 描述 |
|------|------|
| **输入** | `.stylx` 样式文件、图层符号的 CIM JSON |
| **输出** | QGIS 样式 XML 文件、转换报告、错误日志 |
| **输出位置** | 嵌入中间 XML 的样式库元素，同时生成独立 .xml 文件 |

### 4.3 关键算法/流程
1. **.stylx 文件解析**
   - 解析 .stylx（实为 SQLite 数据库）
   - 提取符号项：点、线、面、标注符号
   - 排除 ArcGIS Pro 特有符号效果

2. **符号类型映射**
   - 建立 ArcGIS → QGIS 符号类型映射表：
     - SimpleMarkerSymbol → QgsSimpleMarkerSymbolLayer
     - SimpleLineSymbol → QgsSimpleLineSymbolLayer
     - SimpleFillSymbol → QgsSimpleFillSymbolLayer
   - 处理复杂符号：渐变填充、图片符号等

3. **属性转换**
   - 颜色：RGB/CMYK 转换
   - 大小：单位换算（点→毫米）
   - 线型：实线、虚线、点线映射
   - 填充样式：实心、空心、图案映射

4. **CIM JSON 处理**
   - 解析图层级符号系统的 CIM JSON
   - 提取渲染器信息（单一符号、分类、分级色彩）
   - 转换为 QGIS 对应的渲染器配置

5. **QGIS 样式生成**
   - 按照 QGIS 样式 XML 格式组织
   - 生成符号库、颜色库、线型库
   - 嵌入到中间 XML 的样式库元素

### 4.4 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| .stylx 解析速度 | ≤ 2 秒/文件 | 从打开文件到提取所有符号的时间 |
| 符号转换准确率 | ≥ 95% | 人工抽样验证外观一致性 |
| 内存占用 | ≤ 200 MB | 处理大型样式库时的内存使用 |
| 支持符号类型数 | ≥ 20 种 | 统计成功转换的符号类型 |

### 4.5 异常处理方案
1. **.stylx 文件损坏**
   - 验证文件完整性
   - 尝试恢复或跳过损坏条目
   - 记录详细错误信息

2. **不支持的符号类型**
   - 识别 ArcGIS Pro 特有符号效果
   - 使用默认符号替代
   - 在报告中标记未转换的符号

3. **属性转换异常**
   - 颜色空间转换失败时使用最近似值
   - 单位换算错误时使用默认比例
   - 记录转换异常供人工检查

4. **CIM JSON 解析失败**
   - 验证 JSON 格式
   - 处理缺失字段
   - 提供默认渲染器配置

### 4.6 测试用例
| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 标准 .stylx 文件转换 | 生成完整 QGIS 样式 XML | 在 QGIS 中加载验证 |
| 复杂符号转换 | 保持视觉相似度 ≥ 90% | 截图对比，色差分析 |
| 图层 CIM 符号转换 | 正确生成 QGIS 渲染器 | 在 QGIS 中应用验证 |
| 大型样式库处理 | 在 30 秒内完成转换 | 性能测试，内存监控 |
| 错误样式文件处理 | 优雅失败，生成错误报告 | 检查错误日志和跳过机制 |

## 5. 性能优化模块

### 5.1 功能描述
- 实现多进程架构，按工程或地图/布局拆分任务
- 大表处理：阈值检测自动切换逻辑复制
- 异步任务管理：实现状态跟踪和重试机制
- 资源管理和负载均衡

### 5.2 输入输出定义
| 项目 | 描述 |
|------|------|
| **输入** | 待处理工程列表、系统配置（线程数、超时设置等） |
| **输出** | 优化后的处理流水线、性能报告、资源使用统计 |
| **控制输出** | 任务调度计划、进程状态、错误重试队列 |

### 5.3 关键算法/流程
1. **任务拆分策略**
   - **按工程拆分**：每个 `.aprx` 作为独立任务，适合批量处理
   - **按组件拆分**：单个工程内，地图、布局、数据复制作为子任务
   - **依赖分析**：构建任务依赖图（数据复制 → 样式应用 → 布局重建）

2. **并行处理框架**
   - 使用 `concurrent.futures.ThreadPoolExecutor` 或 `ProcessPoolExecutor`
   - 集成 QgsTask 用于 PyQGIS 环境的并行重建
   - 实现工作窃取（work-stealing）提高负载均衡

3. **大表检测与处理**
   - **阈值检测**：
     a. 查询 `pg_total_relation_size` 估计表大小
     b. 超过配置阈值（默认 100GB）标记为"建议逻辑复制"
   - **逻辑复制流水线**：
     a. 自动配置发布/订阅
     b. 监控初始同步进度
     c. 故障转移和重试机制

4. **异步任务管理**
   - 任务状态机：pending → running → completed/failed
   - 状态持久化：在 XML 中记录任务状态
   - 重试策略：指数退避重试，最大重试次数可配置
   - 超时控制：每个任务设置超时限制（默认 300 秒）

5. **资源管理**
   - 内存监控：防止内存泄漏，设置上限自动清理
   - 连接池管理：数据库连接复用
   - 文件句柄管理：及时释放资源

### 5.4 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 单工程迁移时间 | ≤ 30 秒 | 端到端处理时间 |
| 批量处理吞吐量 | ≥ 5 工程/分钟 | 200 个工程的总体处理时间 |
| CPU 利用率 | ≥ 70% | 监控多核 CPU 使用率 |
| 内存使用效率 | 无泄漏，稳定在阈值内 | 长时间运行测试 |
| 任务成功率 | ≥ 99% | 成功完成的任务比例 |

### 5.5 异常处理方案
1. **进程/线程崩溃**
   - 监控子进程状态
   - 重启崩溃的进程
   - 恢复中断的任务

2. **死锁检测与处理**
   - 监控任务等待时间
   - 实现死锁检测算法
   - 强制终止并重新调度

3. **资源耗尽**
   - 监控内存、CPU、磁盘 I/O
   - 实施背压（backpressure）控制
   - 优雅降级或暂停新任务

4. **网络/数据库波动**
   - 实现弹性重试机制
   - 临时故障的自动恢复
   - 持久化任务状态防丢失

5. **负载不均衡**
   - 动态任务分配
   - 基于工作量的调度
   - 实时监控和调整

### 5.6 测试用例
| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 单工程多线程处理 | 时间比单线程减少 50%+ | 对比测试，性能分析 |
| 200 工程批量处理 | 总时间 ≤ 40 分钟 | 端到端批量测试 |
| 大表逻辑复制触发 | 自动切换复制策略 | 监控任务日志和状态 |
| 资源耗尽场景 | 优雅降级，无崩溃 | 压力测试，监控资源 |
| 任务失败重试 | 自动重试达到成功 | 模拟失败场景测试 |

## 6. 验证系统

### 6.1 功能描述
- 自动化测试脚本对比采样记录与重建结果
- 验证属性、几何、空间参考一致性
- 生成差异报告和迁移质量评估
- 提供回归测试套件

### 6.2 输入输出定义
| 项目 | 描述 |
|------|------|
| **输入** | 中间 XML 文件、重建后的 QGIS 工程、原始采样数据 |
| **输出** | 一致性报告、差异详情、质量评分、可视化对比图 |
| **报告格式** | HTML/PDF 报告、JSON 详细数据、CSV 差异列表 |

### 6.3 关键算法/流程
1. **数据提取**
   - 从中间 XML 读取采样记录
   - 从重建的 QGIS 工程查询对应图层
   - 提取相同的 N 条记录（按 OBJECTID 或其他唯一键）

2. **属性对比**
   - 字段级对比：名称、类型、值
   - 处理数据类型差异：整数/浮点数容差
   - 处理空值/NULL 值
   - 排除 BLOB 字段对比

3. **几何验证**
   - WKT/WKB 格式对比
   - 空间参考一致性验证
   - 几何精度容差处理（浮点误差）
   - 拓扑一致性检查

4. **空间参考验证**
   - 对比 CRS 定义（WKT 或 Proj 字符串）
   - 验证坐标转换一致性
   - 检查投影参数精度

5. **功能测试**
   - 布局导出对比：PDF/图像像素级比较
   - 查询结果一致性：相同 SQL 查询的结果对比
   - 符号渲染验证：视觉对比或符号属性对比

6. **报告生成**
   - 汇总所有不一致项
   - 计算一致性评分
   - 生成可视化差异图
   - 提供修复建议

### 6.4 性能指标
| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 验证执行时间 | ≤ 10 秒/工程 | 从开始验证到报告生成 |
| 检测准确率 | ≥ 99% | 人工验证检测结果 |
| 误报率 | ≤ 1% | 检查错误报告的真实性 |
| 报告完整性 | 100% 覆盖所有差异 | 人工检查报告内容 |

### 6.5 异常处理方案
1. **查询执行失败**
   - 检查 QGIS 工程状态
   - 验证图层可访问性
   - 提供部分验证结果

2. **数据不一致**
   - 分级处理：严重错误 vs 警告
   - 提供差异详情和可能原因
   - 建议修复方案

3. **几何精度误差**
   - 设置合理容差阈值
   - 区分可接受的浮点误差
   - 标记需要人工检查的情况

4. **报告生成失败**
   - 验证输出路径可写
   - 提供替代输出格式
   - 生成简化错误报告

### 6.6 测试用例
| 测试场景 | 预期结果 | 验证方法 |
|----------|----------|----------|
| 完全一致的数据 | 报告显示 100% 一致性 | 检查报告评分和差异列表 |
| 属性值差异 | 准确检测并定位差异字段 | 验证报告中的差异详情 |
| 几何变换错误 | 检测空间参考或几何变形 | 对比 WKT 和可视化检查 |
| 布局像素对比 | 检测渲染差异 ≥ 5 像素 | 图像差分算法验证 |
| 批量验证测试 | 同时验证多个工程，无冲突 | 并发测试，资源监控 |
