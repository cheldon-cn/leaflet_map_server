# 近五年海图/水域图导航系统方向投标信息梳理（含技术参数）

## 概述

导航系统方向近五年投标活动主要围绕船舶导航设备、内河航道智能监测、北斗导航系统、智慧航道平台等展开。采购方以航道管理部门、交通运输局、港口集团、引航站为主，供应商以大连海大智龙、天津路航等为代表。项目金额从不足5万到250万元以上不等，呈现从单导航设备向综合航道感知体系演化的趋势，技术重心从传统雷达/AIS向多传感器融合、人工智能辅助决策方向升级。

## 详细投标信息

### 1. 内河航道船舶智能监测系统 — 技术规格书

- **发布时间**：2022年（技术规范文件）
- **标准编号**：内河航道船舶智能监测系统技术要求
- **适用范围**：内河航道船舶智能监测系统

#### 系统架构与功能要求

**系统总体架构**：

系统由前端感知设备、边缘计算节点、传输网络、系统管理平台和数据存储处理中心组成，实现对内河航道船舶的全面感知与智能监测。

**前端感知设备技术要求**：

| 设备类型 | 技术参数 |
|----------|----------|
| AIS接收器 | 符合ITU-R M.1371建议案；接收灵敏度≤-107dBm；VHF频段：156.025MHz～162.025MHz |
| 雷达 | 作用距离：内河航道0.5～16nm；分辨率：距离分辨率≤20m；方位分辨率≤0.5°；工作频段：X波段或S波段 |
| CCTV摄像头 | 分辨率：1920×1080（Full HD）或以上；低照度：≤0.001Lux；防护等级：IP66 |
| VHF甚高频通信 | 工作频段：156.025MHz～162.025MHz；发射功率：1W/25W可切换；频道数：≥70个国际频道 |
| 激光测距仪 | 测程：≥500m；精度：≤±1m；扫描角度：0°～360° |
| 环境监测传感器 | 风速/风向/能见度/水位等参数采集；数据更新频率：≥1Hz |

**系统管理平台功能要求**：

| 功能模块 | 具体要求 |
|----------|----------|
| 船舶识别 | 自动识别船舶名称、MMSI、船长、船型、载重吨 |
| 船舶跟踪 | 实时跟踪船舶位置、航速、航向；轨迹回放≥30天 |
| 违章检测 | 超速/逆行/违规停泊/桥梁限高等违章自动检测和报警 |
| 统计分析 | 船舶流量统计、拥堵分析、趋势预测 |
| 信息发布 | 通过VHF/短信/APP向船舶推送航行指令和预警信息 |
| 接口要求 | 支持GB/T 28181国标协议；支持数据共享交换 |

**数据存储与传输要求**：

- 数据存储容量：≥6个月历史数据在线存储
- 传输网络：支持光纤/4G/5G/卫星等多种传输方式
- 数据格式：符合交通运输行业相关数据标准

### 2. 内河船舶导航雷达 JT/T 681.1-2024 性能标准

- **标准号**：JT/T 681.1-2024
- **发布日期**：2024年4月2日
- **实施日期**：2024年7月1日
- **代替标准**：JT/T 681.1-2007
- **技术归口**：交通运输信息通信及导航标准化技术委员会
- **主要起草单位**：中国交通通信信息中心、武汉理工大学、集美大学
- **主要起草人**：桑凌志、符强、李军、于树海、马枫、王树武等
- **适用范围**：内河水域船舶所配备导航雷达的设计、生产、检验、安装、操作和维护

#### 核心技术指标变化

相比旧版JT/T 681.1-2007，新版标准主要变化：

| 指标类别 | 旧版（2007） | 新版（2024）变化方向 |
|----------|-------------|-------------------|
| 目标检测能力 | 基本要求 | 提升内河复杂环境下的目标检测灵敏度 |
| 智能化功能 | 无 | 增加自动标绘、目标跟踪、CPA/TCPA计算等智能化要求 |
| 显示模式 | 基本显示 | 增强多种显示模式（RM/TM/N-UP/H-UP）要求 |
| 接口标准 | 基本接口 | 增加网络接口、数据输出标准 |
| 环境适应性 | 基本要求 | 提升内河特殊环境（雾/暴雨/桥梁限高等）的适应性要求 |

## 趋势分析

导航系统方向呈现以下趋势：

**一、从单设备向综合感知体系演进**。传统单一AIS接收器或雷达已无法满足需求，智慧航道建设要求构建"雷达+AIS+VHF+CCTV+激光测距+环境监测"的多传感器融合感知体系。内河航道船舶智能监测系统技术要求（2022）提供了完整的技术规范框架，覆盖从前端感知到后台管理的全链路要求。

**二、内河船舶导航雷达标准全面升级**。JT/T 681.1-2024代替了2007版旧标准，新增智能化功能要求（自动标绘、目标跟踪、CPA/TCPA计算）、多种显示模式要求、网络接口标准，并针对内河特殊环境（雾、暴雨、桥梁限高等）提出更高适应性要求。

**三、北斗导航系统成为刚需**。江苏航运明确采购"船载北斗卫星定位导航系统"，国家标准GB/T 43053-2023明确了兼容北斗的电子海图显示与信息系统的技术要求，北斗+GPS双模成为标准配置。

**四、智慧航道运营服务成为新采购模式**。广东省航道事务中心251万元的"智慧航道系统融合项目系统业务运营服务"，标志航道信息化从一次性建设采购，向持续性运营服务采购模式转型。12个月服务周期、预留中小微企业份额等要求，提供了新型采购范本。

**五、AI辅助决策逐步渗透**。"船舶数智化AI运维辅助决策系统"硬件采购项目的出现，表明人工智能技术已开始进入船舶导航和运维领域，从单纯的位置感知向智能决策支持延伸。

**六、标准化体系日趋完善**。GB/T 46415-2025（AIS A类设备）、GB/T 39620-2025（AIS基站）、DB32/T 4655-2024（内河智慧航道）等标准的密集发布，标志着导航领域进入了规范化、标准化发展的新阶段。
