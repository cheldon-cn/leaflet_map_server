# 海图导航系统需求说明书

## 一、系统概述

### 1.1 系统定位

海图导航系统是海图导航平台的核心功能模块，负责航线规划、航行引导、定位追踪等导航相关功能。系统基于北斗定位信息，结合实时水深、航道条件等多因素进行智能航线规划，辅助船舶安全高效行驶。

### 1.2 系统目标

| 目标 | 具体指标 |
|-----|---------|
| 定位精度 | VIP用户<1m，普通用户5-10m |
| 定位频率 | VIP用户10s/次，普通用户30s/次 |
| 航线规划 | 多因素智能规划，考虑水深、航道宽度、桥梁限高等 |
| 动态避障 | 支持实时障碍物规避 |
| 离线导航 | 支持离线航线规划和导航 |

### 1.3 适用场景

| 场景 | 说明 |
|-----|-----|
| 出航规划 | 出航前制定航行计划 |
| 航行引导 | 航行中实时导航引导 |
| 进港引导 | 进港时精细导航 |
| 应急导航 | 紧急情况下的导航支持 |

### 2.2 NMEA数据解析

#### 2.2.1 NMEA 0183常用语句

| 语句 | 说明 | 示例 |
|-----|-----|-----|
| $GNGGA | 定位数据 | 包含时间、经纬度、定位质量、卫星数等 |
| $GNRMC | 推荐最小定位数据 | 包含时间、状态、经纬度、速度、航向等 |
| $GNVTG | 航迹和地面速度 | 包含航向和速度信息 |
| $GNGSA | 卫星精度因子 | 包含定位模式、卫星PRN、DOP值 |
| $GNGSV | 可见卫星信息 | 包含卫星PRN、仰角、方位角、信噪比 |

#### 2.2.2 GGA语句结构

```
$GNGGA,092750.000,3015.3802,N,12042.2549,E,1,08,1.0,23.5,M,0.0,M,,*7A
```

| 字段 | 内容 | 说明 |
|-----|-----|-----|
| 1 | 092750.000 | UTC时间 09:27:50 |
| 2 | 3015.3802 | 纬度 30°15.3802' |
| 3 | N | 北纬 |
| 4 | 12042.2549 | 经度 120°42.2549' |
| 5 | E | 东经 |
| 6 | 1 | 定位质量(0=无效,1=单点,2=差分) |
| 7 | 08 | 使用卫星数 |
| 8 | 1.0 | HDOP水平精度因子 |
| 9 | 23.5 | 海拔高度 |
| 10 | M | 高度单位(米) |

#### 2.2.3 RMC语句结构

```
$GNRMC,092750.000,A,3015.3802,N,12042.2549,E,0.15,178.68,200524,,,A*60
```

| 字段 | 内容 | 说明 |
|-----|-----|-----|
| 1 | 092750.000 | UTC时间 |
| 2 | A | 状态(A=有效,V=无效) |
| 3 | 3015.3802 | 纬度 |
| 4 | N | 北纬 |
| 5 | 12042.2549 | 经度 |
| 6 | E | 东经 |
| 7 | 0.15 | 地面速度(节) |
| 8 | 178.68 | 航向(度) |
| 9 | 200524 | UTC日期 2024年5月20日 |

### 3.2 航线规划算法

#### 3.2.1 算法选择

| 算法 | 适用场景 | 优缺点 |
|-----|---------|-------|
| A*算法 | 简单场景 | 效率高，但路径可能不够平滑 |
| RRT*算法 | 复杂障碍环境 | 适应性强，但计算量大 |
| 遗传算法 | 多目标优化 | 可优化多个目标，但收敛慢 |
| 蚁群算法 | 动态环境 | 适应动态变化，但参数敏感 |

#### 3.2.2 规划目标

| 目标 | 权重 | 说明 |
|-----|-----|-----|
| 安全性 | 最高 | 避开所有危险区域 |
| 经济性 | 高 | 最短距离/最省油 |
| 时间性 | 中 | 最短航行时间 |
| 舒适性 | 低 | 减少转弯次数 |

#### 3.2.3 约束条件

| 约束 | 说明 |
|-----|-----|
| 最小水深 | 航线水深 >= 吃水 + UKC |
| 最小宽度 | 航道宽度 >= 船宽 × 安全系数 |
| 最大高度 | 桥梁净高 >= 船高 + 安全余量 |
| 禁航区 | 完全避开禁航区域 |
| 转弯半径 | 转弯半径 >= 船舶最小转弯半径 |

### 3.4 UKC计算

#### 3.4.1 UKC定义

UKC（Under Keel Clearance）即富余水深，是指船舶龙骨以下至海底的最小安全距离。

```
UKC = 实际水深 - 船舶吃水
```

#### 3.4.2 UKC组成

| 组成部分 | 说明 | 计算方法 |
|---------|-----|---------|
| 静态UKC | 基础安全余量 | 根据船舶类型确定 |
| 动态UKC | 航行中下沉量 | 根据航速、水深计算 |
| 水位修正 | 潮汐水位变化 | 根据潮汐预报计算 |
| 测量误差 | 水深测量误差 | 根据测量精度确定 |

#### 3.4.3 UKC标准

| 航道类型 | 最小UKC要求 |
|---------|------------|
| 外海航道 | 吃水的20% |
| 港口航道 | 吃水的10% |
| 内河航道 | 吃水的10-15% |
| 受限水域 | 吃水的15-20% |

### 4.3 航行计算

#### 4.3.1 航行参数计算

| 参数 | 计算公式 |
|-----|---------|
| 航行距离 | 大圆距离或恒向线距离 |
| 航行时间 | 距离 / 航速 |
| 燃油消耗 | 时间 × 油耗率 |
| 预计到达时间 | 当前时间 + 航行时间 |

#### 4.3.2 大圆距离计算

```
distance = 2 * R * arcsin(sqrt(sin²((φ2-φ1)/2) + cos(φ1)*cos(φ2)*sin²((λ2-λ1)/2)))
```

其中：
- R = 地球半径（6371km）
- φ1, φ2 = 起终点纬度（弧度）
- λ1, λ2 = 起终点经度（弧度）
