# 执行流程

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、总体流程

```
开始 → 读取tasks.md → 分析优先级 → 选择任务
  ↓
是否有待执行任务？
  ├─ YES → 执行任务 → 更新状态 → 检测阻塞
  │                              ↓
  │                         是否阻塞？
  │                         ├─ YES → 等待用户
  │                         └─ NO → 返回选择任务
  └─ NO → 输出完成报告 → 终止
```

---

## 二、单任务执行流程

| 步骤 | 内容 |
|------|------|
| Step 1 | 任务准备：读取描述、查找设计文档 |
| Step 2 | 测试先行：编写测试用例 |
| Step 3 | 编码实施：实现功能 |
| Step 4 | 测试验证：运行测试、检查覆盖率>80% |
| Step 5 | 状态更新：更新tasks.md状态 |
| Step 6 | 继续检测：扫描下一个任务 |

---

## 三、优先级执行规则

```
P0 (关键/阻塞) → P1 (高) → P2 (中) → P3 (低)
```

**同级任务执行顺序**:
1. 按依赖关系：先执行被依赖的任务
2. 按关键路径：优先执行关键路径上的任务
3. 按里程碑：按M1 → M2 → M3顺序
