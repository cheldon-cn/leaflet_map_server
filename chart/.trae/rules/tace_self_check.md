# 执行自检清单

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、任务完成自检

每个任务完成后必须执行：

```markdown
□ 任务状态已更新为 ✅ Done 或 ❌ Failed
□ 任务状态已写入 tasks.md 文件
□ 里程碑进度已更新
□ 性能数据已收集
□ 已扫描任务清单识别下一个任务
```

---

## 二、继续执行自检

检测是否继续执行：

```markdown
□ 如有待执行任务，已自动继续执行
□ 如无待执行任务，已输出完成报告
□ 如遇到阻塞，已输出阻塞报告
□ 如用户请求停止，已保存进度
```

---

## 三、初始化自检

执行前必须验证：

```markdown
□ .trae/progress/ 目录已创建
□ .trae/logs/ 目录已创建
□ 目录写入权限验证通过
□ 配置文件格式正确
□ 任务清单文件存在
```

---

## 四、自检失败处理

| 情况 | 处理方式 |
|------|----------|
| 状态更新失败 | 重试3次，失败后输出错误 |
| 文件写入失败 | 检查权限，等待用户确认 |
| 配置验证失败 | 使用默认值，输出警告 |
