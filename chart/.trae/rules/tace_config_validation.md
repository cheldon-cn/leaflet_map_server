# 配置验证错误报告

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、错误类型分类

| 错误类型 | 说明 | 示例 |
|----------|------|------|
| 类型错误 | 值类型不匹配 | 字符串应为数字 |
| 范围错误 | 值超出有效范围 | 并发数>5 |
| 缺失字段 | 必需字段未定义 | 缺少limits配置 |
| 格式错误 | 值格式不正确 | 时间格式错误 |

---

## 二、错误报告格式

```markdown
## ⚠️ 配置验证失败

**错误类型**: [类型错误/范围错误/缺失字段/格式错误]
**配置文件**: .trae/config.json

### 错误详情

| 字段路径 | 错误类型 | 当前值 | 期望值 | 修复建议 |
|----------|----------|--------|--------|----------|
| limits.maxTime | 格式错误 | "abc" | "8h" | 使用时间格式 |
```

---

## 三、修复示例输出

```markdown
### 修复示例

**错误字段**: limits.maxExecutionTime
**当前值**: "abc"
**修复后**:
```json
{
  "limits": {
    "maxExecutionTime": "8h"
  }
}
```

---

## 四、验证失败处理

| 情况 | 处理方式 |
|------|----------|
| 配置文件不存在 | 使用默认配置，输出警告 |
| 配置格式错误 | 使用默认配置，输出详细错误 |
| 部分字段错误 | 使用默认值覆盖错误字段 |
