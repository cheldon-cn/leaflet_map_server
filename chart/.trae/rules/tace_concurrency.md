# 并发执行

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、并发判断规则

| 条件 | 说明 |
|------|------|
| 候选任务数 >= 2 | 至少2个任务可并发 |
| 无依赖冲突 | 并发任务间无依赖关系 |
| 资源充足 | 内存、CPU满足要求 |

---

## 二、并发控制机制

| 机制 | 说明 |
|------|------|
| 任务锁 | 防止同一任务被多次执行 |
| 资源限制 | 控制并发任务的总资源消耗 |
| 失败隔离 | 单个任务失败不影响其他 |
| 状态同步 | 并发任务状态更新互不干扰 |
| 依赖检查 | 并发前验证依赖关系 |
| 资源隔离 | 并发任务独立资源上下文 |

---

## 三、并发配置

```json
{
  "concurrency": {
    "enabled": true,
    "maxConcurrent": 3,
    "resourceThreshold": {
      "maxMemory": "1GB",
      "maxCPU": "80%"
    }
  }
}
```

---

## 四、依赖冲突检测

```
IF 任务A依赖任务B AND 任务B在并发列表中 THEN
  → 依赖冲突
  → 解决方案：任务A等待任务B完成后执行
END IF
```
