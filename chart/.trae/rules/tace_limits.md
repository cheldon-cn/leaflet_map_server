# 执行限制机制

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、限制参数

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 最大执行时间 | 8小时 | 防止长时间运行 |
| 最大任务数 | 100个 | 防止任务过多 |
| 最大循环次数 | 1000次 | 防止无限循环 |
| 内存使用上限 | 2GB | 防止内存溢出 |
| 连续失败上限 | 5个 | 防止持续失败 |
| 单任务超时 | 10分钟 | 单个任务最大执行时间 |

---

## 二、配置方式

```json
{
  "limits": {
    "maxExecutionTime": "8h",
    "maxTasks": 100,
    "maxLoops": 1000,
    "maxMemory": "2GB",
    "maxConsecutiveFailures": 5,
    "taskTimeout": "10m"
  }
}
```

---

## 三、超限处理流程

| 步骤 | 内容 |
|------|------|
| Step 1 | 检测是否超限（时间/循环/内存） |
| Step 2 | 保存当前进度到进度文件 |
| Step 3 | 输出超时报告，说明原因 |
| Step 4 | 等待用户确认继续或终止 |

---

## 四、连续失败处理

```markdown
IF 连续失败任务数 >= 最大连续失败上限 THEN
  → 输出失败报告
  → 分析失败原因
  → 等待用户确认是否继续
END IF
```
