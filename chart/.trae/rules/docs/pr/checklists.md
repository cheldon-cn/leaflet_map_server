# 完整检查清单

> **详细文档**: @doc/experience_standard.md

---

## 一、编码前检查

```markdown
□ 确认项目C++标准版本为C++11 [STD-01~05]
□ 查阅相关模块的index.md了解已有类和方法，避免重复实现
□ 确认API命名规范（GetSize而非Size）[API-01~03]
□ 确认设计文档中对应的模块接口定义
□ 确认所有需要的头文件 [HDR-01~04]
□ 确认DLL导出宏配置 [DLL-01~02]
```

---

## 二、编码时检查

```markdown
□ 所有使用的标准库类型都有对应的#include [HDR-01]
□ const方法中调用的方法都是const的 [CONST-01]
□ 需要在const方法中修改的成员声明为mutable [CONST-02]
□ 智能指针转换使用正确的方式 [PTR-01~05]
□ 纯虚接口类提供了工厂方法 [INT-01]
□ 使用override关键字确保虚函数覆盖 [INT-03]
□ 成员变量在构造函数初始化列表中初始化 [INIT-01]
```

---

## 三、编译前检查

```markdown
□ CMakeLists.txt中源文件列表已更新
□ 输出目录配置正确（包括配置特定变量）[CMAKE-01]
□ DLL导出宏配置正确 [DLL-01~02]
□ 链接库配置正确 [CMAKE-02]
□ 测试目标链接了被测模块 [TEST-01]
```

---

## 四、测试前检查

```markdown
□ 测试用例使用正确的API方法名 [API-01~03]
□ 测试目标链接了被测模块 [TEST-01]
□ 抽象基类创建了派生类进行测试 [UT-02]
□ 测试边界条件和错误处理
□ 并发测试已禁用或特殊处理
```

---

## 五、任务执行增强规则

当用户发送"按任务清单tasks.md执行"时，除了遵守上述所有规则外，还须：

- 每开始一个新任务前，执行"编码前检查"
- 每完成一个任务后，执行"编码后自检"（编码时检查、编译前检查）
- 如果自检未通过，**不得标记任务为完成**，而是先修正问题
- 在任务状态更新时，附加一句："已通过编码规范自检"
- 遇到问题时，查阅 @doc/experience_standard.md 获取详细解决方案
