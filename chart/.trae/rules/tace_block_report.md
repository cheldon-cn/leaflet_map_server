# 阻塞报告格式

> **主规则**: @.trae/rules/task_continuous_execution.md

---

## 一、阻塞判断标准

```markdown
IF 问题无法自动解决 AND 需要外部资源 THEN
  → 识别为阻塞问题
  → 输出阻塞报告
  → 等待用户确认
END IF
```

---

## 二、报告格式

```markdown
## ⚠️ 阻塞问题

### 任务ID: T1 - 任务名称

**阻塞原因**:
- 详细描述阻塞原因

**影响范围**:
- 列出受影响的任务

**建议解决方案**:
- 提供解决方案选项

**等待用户确认**:
- 请选择解决方案或提供其他指示
```

---

## 三、阻塞类型

| 类型 | 说明 |
|------|------|
| 依赖缺失 | 缺少库、工具或环境 |
| 编译失败 | 代码编译或构建失败 |
| 需求不明确 | 任务描述模糊或矛盾 |
