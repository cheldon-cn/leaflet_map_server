# Project Initializer Skill

## 概述

此技能用于初始化新的项目，提供完整的项目结构、技术栈配置、Git仓库和基础框架搭建。支持**全新创建**、**增量更新**和**设计文档驱动**三种模式。

## 使用场景

- 创建新项目
- 初始化项目结构
- 设置特定技术栈
- 项目脚手架搭建
- **增量更新现有项目**
- **根据设计文档生成代码**

## 支持的技术栈

| 技术栈 | 描述 |
|--------|------|
| C++ Core + Java Bridge | C++核心库 + JNI桥接 + Java封装 + Spring Boot应用 |
| Pure Java | 纯Java项目，支持Maven/Gradle |
| Pure C++ | 纯C++项目，使用CMake |
| Hybrid | 自定义技术组合 |

## 三种操作模式

### Mode 1: 全新创建
- 目标目录为空或用户明确要求创建新项目
- 完整创建所有目录、文件、Git仓库和基础框架

### Mode 2: 增量更新
- 目标目录已存在项目
- 智能检测缺失模块并补充
- 不重复初始化Git仓库
- 不覆盖已有文件（除非用户确认）

### Mode 3: 设计文档驱动
- 根据用户提供的设计文档生成代码结构
- 解析Markdown/Word/PDF文档
- 自动提取类、接口、配置信息

## 工作流程

```
Phase 0: 环境检测
├── 检查目标目录是否存在
├── 检测项目指示器（.git, CMakeLists.txt, pom.xml等）
├── 分析现有结构
└── 确定操作模式

Phase 1: 需求收集
├── 全新项目：询问项目名称、技术栈等
├── 增量更新：展示检测结果，询问补充内容
└── 设计驱动：询问文档路径和生成选项

Phase 2: 文件冲突处理
├── 文件不存在 → 直接创建
├── 文件存在，内容相同 → 跳过
├── 文件存在，内容不同 → 询问处理方式
└── 配置文件 → 智能合并

Phase 3: Git仓库处理
├── 无.git → 询问是否初始化
└── 有.git → 跳过

Phase 4: 基础框架检测与搭建
├── 检测各组件是否存在
└── 仅创建缺失的组件

Phase 5: 设计文档解析（可选）
├── 解析章节结构
├── 提取代码块
└── 生成代码文件

Phase 6-8: 创建结构、组件、验证
```

## 文件冲突处理规则

| 场景 | 处理方式 |
|------|----------|
| 文件不存在 | 直接创建 |
| 文件存在，内容相同 | 跳过，无需操作 |
| 文件存在，内容不同 | 询问用户处理方式 |
| 用户明确要求覆盖 | 备份后覆盖 |

## Git仓库处理规则

| 条件 | 操作 |
|------|------|
| 目录为空 | 初始化Git仓库 |
| 目录存在，无.git | 询问是否初始化 |
| 目录存在，有.git | 跳过，不重复初始化 |

## 基础框架组件检测

| 组件 | 检测文件 | 创建条件 |
|------|----------|----------|
| 异常处理 | `exception.hpp` / `CoreException.java` | 文件不存在时创建 |
| 日志系统 | `logger.hpp` / `logback.xml` | 文件不存在时创建 |
| 数据库连接池 | `DatabaseConfig.java` | 需要数据库且文件不存在 |
| 健康检查 | `HealthController.java` | Web服务且文件不存在 |
| 配置管理 | `application*.yml` | 文件不存在时创建 |

## 模板文件

| 文件 | 用途 |
|------|------|

## 重要规则

### 文件处理规则
1. **不询问不覆盖** - 始终先检查文件是否存在
2. **覆盖前备份** - 用户确认覆盖时创建.bak文件
3. **配置文件智能合并** - 保留用户自定义内容
4. **相同文件跳过** - 不处理与模板相同的文件

### Git规则
1. **初始化前检查** - 不重复初始化已有Git仓库
2. **新项目询问** - 新目录确认是否初始化Git
3. **已有则跳过** - 存在.git时自动跳过

### 框架规则
1. **创建前检测** - 检查组件是否已存在
2. **仅添加缺失** - 不重建已有框架组件
3. **保留自定义** - 不覆盖用户修改的文件

### 设计文档规则
1. **解析结构** - 从文档提取模块、类、接口
2. **生成桩代码** - 创建占位实现
3. **链接设计** - 添加注释引用设计文档章节

## 注意事项

- 使用绝对路径创建文件
- 尊重现有文件，覆盖前询问
- 根据用户语言偏好生成注释
- 包含完善的错误处理
- 遵循命名约定
- 支持中英文交互
