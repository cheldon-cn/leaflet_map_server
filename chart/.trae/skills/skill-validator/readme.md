# Skill Validator 使用说明

## 概述

skill-validator 是一个用于评估其他 Skill 质量的工具，从六个维度进行全面评估，生成详细的评估报告。

## 触发条件

当出现以下情况时，应调用本 Skill：

- 用户需要对 skill 进行质量评估
- 用户准备发布 skill 但不确定质量是否达标
- 用户询问 skill 质量、评估、验证等关键词
- 用户提到"skill评估"、"质量检查"、"验证skill"等

## 使用方法

### 基本用法

```
请评估 [skill-name] 的质量
你是高质量skill的高级开发者和使用者，对高质量skill有独特理解和实践，开发多款skill风靡技术社区获得百万赞誉；
```

或

```
检查 .trae/skills/[skill-name]/SKILL.md 的质量
```

### 评估输出

评估完成后，会生成以下文件：

```
.trae/skills/[skill-name]/[skill-name]-skill-validator-result.md
```

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 功能维度 | 25% | 核心功能实现程度 |
| 文档维度 | 20% | 文档完整性和可读性 |
| 可用性维度 | 20% | 易用性和引导性 |
| 可靠性维度 | 15% | 错误处理和健壮性 |
| 性能维度 | 10% | 效率和可扩展性 |
| 可维护性维度 | 10% | 代码质量和可维护性 |

## 质量等级

| 等级 | 得分范围 | 说明 |
|------|----------|------|
| 🟢 S级（卓越） | 95-100 | 各维度表现卓越，可作为标杆 |
| 🟢 A级（优秀） | 85-94 | 各维度表现优秀，可直接使用 |
| 🟡 B级（良好） | 75-84 | 整体良好，有少量改进空间 |
| 🟠 C级（合格） | 60-74 | 基本可用，需要改进 |
| 🔴 D级（不合格） | <60 | 存在严重问题，需要重大改进 |

## 评估报告内容

评估报告包含以下部分：

1. **评估结果汇总** - 质量等级、维度得分、问题统计
2. **各维度详细评估** - 每个维度的详细评估结果
3. **问题详情** - 所有问题的详细记录，包括：
   - 问题描述
   - 问题原因
   - 影响分析
   - 修改建议
   - 修改前后对比
   - 问题状态
4. **改进建议** - 按优先级排列的改进建议
5. **评估结论** - 总体评价和改进方向
6. **迭代历史记录** - 迭代修改的历史记录（如有迭代）

## 迭代修改机制

### 自动迭代规则

| 条件 | 操作 |
|------|------|
| 综合得分 < 96分 | 自动进入迭代修改流程，根据评估报告改进Skill |
| 综合得分 ≥ 96分 | 输出提示："🎉 Skill很优秀，可以直接使用了！" |

### 迭代限制

- **单轮对话最大迭代次数**: 6次
- **迭代触发条件**: 得分 < 96
- **迭代终止条件**: 得分 ≥ 96 或 达到最大迭代次数

### 迭代流程

```
评估 → 得分检查 → 
  ├─ 得分 ≥ 96 → 🎉 "Skill很优秀，可以直接使用了！"
  └─ 得分 < 96 → 迭代修改 → 重新评估 → 循环（最多6次）
```

### 迭代终止处理

- **达标终止**: 输出成功提示，Skill可直接使用
- **达到最大迭代次数**: 输出当前结果，建议人工介入优化
