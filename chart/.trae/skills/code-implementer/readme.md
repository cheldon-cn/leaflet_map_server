# Code Implementer Skill 使用说明

## 概述

`code-implementer` 是一个用于指导代码实施的技能，采用TDD（测试驱动开发）流程，确保代码质量和测试覆盖率。

## 触发方式

### 自动触发

当用户请求包含以下关键词时，系统会自动调用本技能：

| 中文关键词 | 英文关键词 |
|------------|------------|
| 编码实施 | implement tasks |
| 任务清单 | follow tasks.md |
| tasks.md | start coding |
| 按照清单 | task list |
| 继续实施 | - |
| 里程碑 | - |
| 实现任务 | - |

### 触发示例

**会触发技能的请求：**
```
按照code\graph\doc\tasks.md清单进行编码实施
继续按照tasks.md清单实施
实施下一个任务
完成里程碑M1的任务
开始编码实现
```

**不会触发技能的请求：**
```
查看任务清单（只是查看，不是实施）
任务清单是什么（询问性质）
帮我规划任务（应触发task-planner）
```

## 核心功能

### 1. 任务分析与优先级

- 自动识别核心框架任务
- 确定任务依赖顺序
- 制定实施计划

### 2. TDD开发流程

- 测试先行（红）
- 编写最小实现（绿）
- 重构优化

### 3. 自动状态更新（重要）

**⚠️ 以下行为会在编码实施过程中自动执行：**

1. **任务状态自动更新**：每完成一个任务，立即更新tasks.md中的任务状态
2. **里程碑状态自动更新**：每完成一个里程碑，立即更新tasks.md中的里程碑状态和汇总统计
3. **进度统计自动更新**：同步更新Overview中的完成数、剩余数、进度百分比

### 4. 质量保证

- 测试覆盖率 >80%
- 代码审查
- 文档同步

## 使用流程

```
用户请求 → 触发code-implementer → 分析任务清单 → 按优先级实施
                                        ↓
                               完成任务 → 更新tasks.md状态
                                        ↓
                               检查里程碑 → 更新里程碑状态
                                        ↓
                               继续下一个任务
```

## 常见问题

### Q1: 为什么我的请求没有触发code-implementer？

**可能原因：**
1. 请求中不包含触发关键词
2. 请求是询问性质而非实施性质
3. 请求应该触发其他技能（如task-planner）

**解决方法：**
确保请求包含"编码实施"、"任务清单"、"tasks.md"等关键词，并明确表示要开始实施。

### Q2: 为什么任务完成后tasks.md没有更新？

**可能原因：**
1. 技能未被正确触发
2. 编译或测试失败导致任务未完成

**解决方法：**
确保技能被正确触发，并检查编译和测试结果。

### Q3: 如何查看当前进度？

技能会在每次任务完成后自动更新tasks.md中的进度统计，包括：
- Completed Tasks
- Remaining Tasks
- Current Progress
